# Cryptology_labs

## 1) Реалізувати алгоритм цифрового підпису DSA

Програмна реалізація алгоритму DSA базується на створенні цифрового підпису для повідомлення та перевірці його справжності за допомогою відкритого ключа. Алгоритм дозволяє підтвердити автора повідомлення та перевірити, чи не були дані змінені після підписання.

Програма складається з таких основних етапів:

* **Генерація ключів:** Спочатку програма створює пару ключів DSA: приватний та публічний. Приватний ключ використовується для створення цифрового підпису, а публічний ключ - для його перевірки.

* **Створення цифрового підпису:** Користувач вводить повідомлення. Програма застосовує алгоритм `SHA256withDSA`, обробляє повідомлення та формує цифровий підпис за допомогою приватного ключа.

* **Перевірка підпису:** Програма перевіряє підпис за допомогою публічного ключа. Якщо повідомлення не змінювалося після підписання, результат перевірки буде успішним.

* **Перевірка зміни повідомлення:** Для демонстрації роботи алгоритму програма змінює початкове повідомлення та повторно перевіряє підпис. Після зміни тексту підпис стає недійсним.

* **Результат:** На екран виводиться початкове повідомлення, публічний ключ, приватний ключ, цифровий підпис у форматі `Base64` та результат перевірки.

## 2) Реалізувати алгоритм 3DES

Програмна реалізація алгоритму 3DES базується на симетричному блоковому шифруванні. Алгоритм 3DES є розвитком DES і застосовує шифрування DES тричі, що підвищує стійкість порівняно зі звичайним DES.

Програма складається з таких основних етапів:

* **Підготовка ключа:** Користувач вводить текстовий ключ. Для роботи алгоритму 3DES формується ключ довжиною 24 байти. Якщо введений ключ коротший, його байти повторюються до потрібної довжини.

* **Шифрування повідомлення:** Користувач вводить відкритий текст. Програма використовує алгоритм `DESede/ECB/PKCS5Padding` і шифрує повідомлення за допомогою сформованого ключа.

* **Кодування шифротексту:** Оскільки зашифровані байти можуть містити нечитабельні символи, результат шифрування переводиться у формат `Base64`.

* **Розшифрування повідомлення:** Програма декодує шифротекст із формату `Base64` та розшифровує його тим самим ключем. Оскільки 3DES є симетричним алгоритмом, для шифрування і розшифрування використовується один ключ.

* **Результат:** На екран виводиться відкритий текст, ключ, використаний алгоритм, шифротекст у форматі `Base64` та розшифрований текст.
